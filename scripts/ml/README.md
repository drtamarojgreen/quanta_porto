# Machine Learning Pipeline: Human vs. LLM Text Detection

This directory implements a hybrid detection system that balances high-accuracy neural discrimination with interpretable stylometric analysis. It is designed to identify whether a text was authored by a human or generated by a Large Language Model (LLM).

## Core Features

The pipeline extracts 24 interpretable features across four categories to capture "authorship style" independent of topic:

### 1. Stylometric & Lexical Features
- **TTR (Type-Token Ratio)**: Lexical diversity (unique words / total words).
- **Hapax Legomena Ratio**: Ratio of words that appear only once (indicates vocabulary richness).
- **Average Word Length**: Mean character count per word.
- **Sentence Length (Mean & Std)**: Captures rhythmic patterns in writing.
- **POS Distributions**: Frequency of Nouns, Verbs, Adjectives, Adverbs, Pronouns, Adpositions, and Conjunctions.
- **Function Word Ratio**: Reliance on "glue words" (the, and, in, etc.), which often differs between humans and LLMs.

### 2. Syntactic Features
- **Passive Voice Ratio**: Frequency of passive constructions (`nsubjpass`), a common hallmark of certain LLM default styles.

### 3. Sentiment & Affective Features
- **Sentiment Scores (VADER)**: Mean and Standard Deviation of Compound, Positive, Negative, and Neutral scores. This detects "emotional consistency" vs. "human volatility."

### 4. Discourse & Cohesion
- **Entity Density**: Average number of unique named entities per sentence.
- **Noun Diversity**: Ratio of unique nouns to total nouns, tracking how "focused" the discourse is.

## How to Use

### 1. Installation
```bash
pip install -r requirements.txt
python -m spacy download en_core_web_sm
```

### 2. Training the Pipeline
1. **Prepare Data**: Create human and LLM datasets with `prompt` and `text` columns.
2. **Execute Training**:
   ```python
   from data_prep import balance_and_split_data
   from features import extract_all_interpretable_features
   from pipeline import train_interpretable_model, train_transformer_model

   # 1. Match and split
   train_df, val_df, test_df = balance_and_split_data(human_df, llm_df)

   # 2. Extract Features (Interpretable)
   X_train = extract_all_interpretable_features(train_df['text'].tolist())
   X_test = extract_all_interpretable_features(test_df['text'].tolist())

   # 3. Train RF Model
   rf_model, scaler = train_interpretable_model(X_train, train_df['label'], X_test, test_df['label'], feature_names)

   # 4. Fine-tune Transformer (Requires GPU recommended)
   trainer, tokenizer = train_transformer_model(train_df['text'], train_df['label'], test_df['text'], test_df['label'])
   ```

### 3. Inference & Explanation
Use the `hybrid_predict` function. If the neural model is uncertain, it falls back to the RF model and flags that an explanation is needed.
```python
from evaluate_explain import explain_interpretable_model
# Generate SHAP plots for fallback cases
explain_interpretable_model(rf_model, X_test, feature_names)
```

## spaCy Resource Requirements

The pipeline defaults to `en_core_web_sm` for efficiency. Depending on your accuracy needs and hardware, you may swap models:

| Model Name | Size on Disk | Memory (RAM) | Description |
|------------|--------------|--------------|-------------|
| **en_core_web_sm** | ~12 MB | ~50 MB | Fast, efficient, uses context-independent vectors. Best for quick stylometry. |
| **en_core_web_md** | ~40 MB | ~500 MB | Medium size, includes 20k real word vectors for better semantic similarity. |
| **en_core_web_lg** | ~560 MB | ~1.5 GB | Large, includes 685k word vectors. Most accurate for CPU-based dependency parsing. |
| **en_core_web_trf**| ~438 MB | ~3 GB+ | Transformer-based (RoBERTa). Extremely accurate but very slow on CPU; GPU highly recommended. |

**Disk Space Recommendation**: Ensure at least **1 GB** of free space if you intend to experiment with the `lg` (large) models.
**Note**: Feature extraction is batched (`batch_size=256`) in `features.py` to optimize memory usage during large-scale processing.

## Triple-Graph Comparative Engine (`corpus_analysis.py`)

This professional engine enables simultaneous analysis and comparison of three co-occurrence networks: **Human**, **LLM**, and **Reference Ontology**. It uses pure graph-theoretic topology to identify structural deviations between domains.

### Core Features

1.  **Triple-Way Comparison**: Contrast human and LLM outputs against a ground-truth ontology (induced from a reference corpus).
2.  **Weighted Nodes**: Assign custom weights to specific keywords in your `dimensions.json` to influence global and local centrality metrics.
3.  **Ontology Induction**: Extract a "clean" reference graph from a corpus using strict PPMI pruning (controlled via `--ont_threshold`).
4.  **Mathematical Rigor**: No linguistic heuristics; metrics are calculated using linear algebra (`eigen_centrality`, `clustering_coefficient`, etc.).

### How to Use

1.  **Define Dimensions and Weights**: Configure your target nodes and their respective weights in `dimensions.json` (see `dimensions.json.template`).
2.  **Run Comparison**:
    ```bash
    python scripts/ml/corpus_analysis.py \
      --human human_text.txt \
      --llm llm_generated.txt \
      --ontology technical_manual.txt \
      --config dimensions.json \
      --ont_threshold 1.5 \
      --output comparative_report.csv
    ```
3.  **Analyze**: The output `comparative_report.csv` contains metrics prefixed with `human_`, `llm_`, and `ont_` for each of your defined dimensions.

### Supported Topology Metrics

- `eigen_centrality`: Global influence of nodes, adjusted by custom weights.
- `degree_centrality`: Local connectivity strength, adjusted by custom weights.
- `clustering_coefficient`: Measures triadic closure and thematic cohesion.
- `associative_strength`: Raw mean association strength (PPMI).

## Component Overview

- `features.py`: The extraction engine (Stylometric, Passive Voice, Sentiment, Entities).
- `data_prep.py`: Logic for prompt-matching and balancing datasets to avoid topically-biased models.
- `pipeline.py`: Training wrappers for Random Forest and HuggingFace Transformers.
- `evaluate_explain.py`: SHAP-based interpretability and hybrid system metrics.

## Feature Mapping

The interpretable model utilizes a 24-dimension vector:
- `[0:13]`: Lexical/Stylometric (TTR, Hapax, Word/Sent lengths, POS counts, Function words).
- `[13]`: Passive Voice Ratio.
- `[14:21]`: Sentiment (Compound, Pos, Neg, Neu - Mean & Std).
- `[22:23]`: Discourse (Entity Density, Noun Diversity).
